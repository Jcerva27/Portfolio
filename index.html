Connecting to API

The focus of this project was to create a script that would automatically pull data from Coinmarketcap's API.
The user gets to choose how many times the data will be pulled and how long the time interval is between each pull. So if you wanted to gather the price of Bitcoin for the whole week, you can specify that you want to pull the data 7 times, with a time interval of 1440 minutes between each pull.
After the data gathering is complete the script will then clean certain problems the data had, then store it as a Csv File.
The purpose of creating this is that in order to get a range of dates and their closing price for that day, you would need to pay in order to use that specifc API point. By Automating the pull request, you can essentially get the daily price of certain Crypto currencies for a certain time period for free.

Analyzing the Movie Industry

The data for this project was downloaded from Kaggle.
In this project I try to demonstrate the logic I use to clean the data and tackle certain problems that arise.
One of the problems I faced with this dataset was that there was about 30% of the data missing for the budget column.
Due to the large amount of missing data, I tried two methods of handling the missing data:

  • Removing the NAN Values and analyzing the remaining data
  • Creating a predictive model that fills in the missing values

There are 4 major sections in this project:
Data Cleaning, Visualization, creating a Linear Regression Model, and Creating a Csv/Excel File.
